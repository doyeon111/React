1. React란?
- Meta(facebook)이 개발한 오픈소스 JavaScript 라이브러리
- 대규모 웹 서비스의 UI를 더 편하게 개발하기 위해 만들어진 기술
- ex) 넷플릭스, 페이스북, 인스타, 노션 등

* 기술적인 특징
1) 컴포넌트를 기반으로 UI를 표현함. (컴포넌트 = 화면을 구성하는 요소)

2) 화면 업데이트 구현이 쉬움. (선언형 프로그래밍 = 과정은 생략하고 목적만 간결히 명시하는 방법) <-> (명령형 프로그래밍 = 모든 과정을 하나하나 다 설명, 코드가 길고 복잡)
    - 목적만 깔끔하게 명시
    - 코드가 간결함
    -> 업데이트를 위한 복잡한 동작을 직접 정의할 필요 없이 특정 변수의 값을 바꾸는 것 만으로도 화면을 업데이트 시킬 수 있다.

3) 화면 업데이트가 빠르게 처리됨.
    - 브라우저의 렌더링 과정
    HTML -> DOM(요소들의 위치, 배치, 모양에 관한 모든 정보) -> Render Tree -> Layout(요소의 배치를 잡는 작업) -> Painting(실제로 화면에 그려내는 과정)
    CSS  -> CSSOM(요소들의 스타일과 관련된 모든 정보)       -> Render Tree -> Layout(요소의 배치를 잡는 작업) -> Painting(실제로 화면에 그려내는 과정)

    - JavaScript가 DOM을 수정하면 업데이트가 발생함.
    JavaScript -> DOM -> Render Tree -> Layout -> Painting (수정 시 Layout과 Painting은 굉장히 오래걸림)

    * 효율적인 수정 방법
    - 다양한 업데이트 -> 동시에 발생한 업데이트를 모음 -> 다 모았다면 한번에 수정 -> DOM
    ==> React는 이 과정을 자동으로 해줌!! (Feat. Virtual DOM)
